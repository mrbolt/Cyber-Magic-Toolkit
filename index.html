Encoding / Decoding

🔤 Base64

How does this work? (Explain like I'm 5)

Imagine you have a secret toy, but you can only send letters to your friend. Base64 is like turning your toy into a description using only letters (A-Z, a-z), numbers (0-9), and two special characters (+ and /). Your friend reads the letters and rebuilds the toy! Computers use Base64 to send things like images through text-only channels (like email).

Base64 is NOT encryption — anyone can decode it. It's just a way of representing data in a text-safe format. Every 3 bytes of data becomes 4 Base64 characters.

🔢 Hexadecimal

How does this work? (Explain like I'm 5)

You know how we count with 10 fingers (0–9)? Hexadecimal counts with 16 "fingers" — it uses 0–9 AND the letters A–F. So after 9, instead of 10, we say A, then B, C, D, E, F, and THEN 10! Computers love hex because each hex digit represents exactly 4 bits, making it a neat shorthand for binary.

Hex is often used in CTFs to represent memory addresses, color codes (#FF00AA), and raw data. Look for "0x" prefix — that usually means hex!

🌐 URL Encoding

How does this work? (Explain like I'm 5)

Web addresses (URLs) can only use certain characters. If you want to put a space or special symbol in a URL, you replace it with a percent sign (%) followed by a number. For example, a space becomes %20. It's like giving every special character a secret code name so the internet can understand it!

URL encoding (also called "percent encoding") is common in web CTF challenges. Watch for double-encoding tricks where %25 = % itself!

💻 Binary

How does this work? (Explain like I'm 5)

Computers only understand two things: ON and OFF — like a light switch! We write ON as 1 and OFF as 0. Every letter, number, and emoji you see is secretly made of 1s and 0s inside the computer. The letter 'A' is 01000001 in binary — that's 8 switches in a row!

Each 0 or 1 is called a "bit." Eight bits make a "byte." Every character you type is usually 1 byte (8 bits) in ASCII. In CTFs, you might get a message as a long string of 1s and 0s!
Encoding / Decoding

🔄 ROT13

How does this work? (Explain like I'm 5)

Imagine the alphabet written in a circle. ROT13 means "rotate by 13 places." Every letter gets replaced by the letter 13 spots ahead. Since there are 26 letters, doing ROT13 twice gets you back to the original! A→N, B→O, C→P... It's like a secret decoder ring!

ROT13 is its own inverse — encoding and decoding use the exact same operation! It's super common in CTFs as a first layer of obfuscation. Numbers and special characters stay the same.

📊 ASCII

How does this work? (Explain like I'm 5)

Computers don't understand letters — they only understand numbers! So humans made a table where every letter gets a number. 'A' = 65, 'B' = 66, 'a' = 97, '0' = 48, space = 32. This table is called ASCII. When you press a key, the computer actually sees a number!

ASCII stands for "American Standard Code for Information Interchange." It defines 128 characters (0–127). In CTFs, you'll often need to convert between decimal numbers and text characters.

📡 Morse Code

How does this work? (Explain like I'm 5)

Imagine you can only make two sounds — a short beep (dot: .) and a long beep (dash: -). Morse Code turns every letter into a pattern of dots and dashes. 'S' is three short beeps (. . .) and 'O' is three long beeps (- - -). That's how "SOS" became the famous distress signal!

Morse Code was invented in the 1830s for telegraphs. In CTFs, you might find messages hidden as dots and dashes, or even as short/long flashes of light or sound beeps!

📝 HTML Entities

How does this work? (Explain like I'm 5)

In web pages, some characters are special — like < and > which tell the browser "this is code!" If you want to actually SHOW a < symbol on the page, you write &lt; instead. It's like saying "I mean the actual character, not a command!" These special codes are called HTML entities.

HTML entities are crucial in web security! XSS (Cross-Site Scripting) attacks happen when special characters aren't properly encoded. CTFs often involve encoding/decoding HTML entities to find hidden payloads.

🏛️ Caesar Cipher

How does this work? (Explain like I'm 5)

Julius Caesar (the famous Roman emperor) used this trick to send secret messages to his generals! He shifted every letter by 3 places: A→D, B→E, C→F. So "HELLO" becomes "KHOOR." If you know the shift number (the "key"), you can shift back to read the message. It's like a secret decoder ring from a cereal box!

The Caesar Cipher is a "substitution cipher" — each letter is substituted for another. It's easy to break because there are only 25 possible shifts! Try the "Brute Force" button to see all possibilities at once.

🔑 Vigenère Cipher

How does this work? (Explain like I'm 5)

Caesar Cipher uses ONE shift for all letters, which makes it easy to crack. Vigenère is smarter — it uses a SECRET WORD as the key! Each letter of the key tells you how much to shift. If your key is "CAT," the first letter shifts by C(2), second by A(0), third by T(19), then it repeats: C, A, T, C, A, T... It's like having a different decoder ring for each letter!

The Vigenère cipher was considered "unbreakable" for 300 years until Friedrich Kasiski cracked it in 1863. The trick? Look for repeating patterns in the ciphertext to figure out the key length.

🛡️ AES Encryption

How does this work? (Explain like I'm 5)

AES is like putting your message in an ULTRA-STRONG safe with a password. You put your secret message in, lock it with a password, and out comes total gibberish. Only someone with the EXACT same password can unlock it. AES is so strong that even the world's fastest computers can't crack it — governments and banks use it every day to protect secrets!

AES (Advanced Encryption Standard) is "symmetric" encryption — the same key encrypts AND decrypts. It works in "rounds" — scrambling data 10-14 times with math operations. It's the gold standard of modern encryption (used in HTTPS, WiFi, file encryption).

⊕ XOR Cipher

How does this work? (Explain like I'm 5)

XOR is a simple rule: "Are these two things DIFFERENT?" If yes → 1, if no → 0. Imagine two light switches — if they're both up or both down, the light is OFF (0). If one is up and one is down, the light is ON (1). To encrypt, you XOR your message with a key. The magic? XOR it with the same key again, and you get the original message back!

XOR is EXTREMELY common in CTFs! Many real-world encryption algorithms (including AES) use XOR as a building block. Single-byte XOR is easy to crack with frequency analysis, but repeated-key XOR with a long key can be tough.

🚂 Rail Fence Cipher

How does this work? (Explain like I'm 5)

Imagine writing your message in a zigzag pattern, like a roller coaster going up and down across several rows (rails). "HELLO WORLD" with 3 rails looks like a zigzag: H goes on top, E goes middle, L goes bottom, L goes middle, O goes top... Then you read each rail left to right! The letters get all mixed up, but they're all still there — just scrambled!

The Rail Fence is a "transposition cipher" — it doesn't change letters, just rearranges them. It's different from Caesar/Vigenère which are "substitution ciphers." In CTFs, try different numbers of rails (2-10) to crack it!
Steganography

🖼️ Image LSB Steganography

How does this work? (Explain like I'm 5)

Every pixel in an image has a color made of Red, Green, and Blue numbers (0-255). The LAST bit of each number is so tiny, changing it doesn't change how the picture looks to your eyes! So we can secretly swap those tiny bits with our secret message bits. The image looks exactly the same, but hidden inside is a secret! It's like whispering a secret so quietly that nobody can hear it, but your friend with super hearing can!

LSB = "Least Significant Bit." Each color channel (R,G,B) has 8 bits, and we modify only the last one. This means each pixel can hide 3 bits of data (one per channel). For a 1000x1000 image, that's ~375KB of hidden data! Tools like "stegsolve" and "zsteg" are popular in CTFs.

👻 Text Steganography

How does this work? (Explain like I'm 5)

Imagine writing an invisible message between the words of a normal letter! Text steganography hides secret messages inside regular text using invisible characters. These "zero-width" characters are like invisible ink — you can't see them, but the computer can read them! The text looks perfectly normal, but there's a hidden message riding along!

Zero-width characters (ZWC) include: Zero-Width Space (U+200B), Zero-Width Non-Joiner (U+200C), and Zero-Width Joiner (U+200D). Each secret bit is encoded as one of these invisible characters. They're often found in CTF challenges hidden in seemingly normal text!
Steganography / Forensics

📸 EXIF / Metadata Viewer

How does this work? (Explain like I'm 5)

When you take a photo with a camera or phone, it secretly writes extra information INTO the picture file — like a hidden diary! This "metadata" can include when the photo was taken, what camera was used, and even WHERE you were standing (GPS coordinates)! EXIF is the name for this hidden diary. In CTFs, secrets are often hidden in this metadata!

EXIF = "Exchangeable Image File Format." Metadata can contain: camera model, date/time, GPS coordinates, software used, comments, copyright info, and custom fields. CTF flags are often hidden in "Comment," "Artist," or "UserComment" fields. Tools like "exiftool" are essential for forensics!

🔍 File Header Analysis

How does this work? (Explain like I'm 5)

Every file starts with special "magic bytes" — like a name tag that says "I'm a PNG image!" or "I'm a ZIP file!" Sometimes in CTFs, sneaky people CHANGE the name tag to trick you. A file might say it's a .txt, but the magic bytes inside say it's actually a .png image! File header analysis reads these magic bytes to reveal the TRUE identity of any file.

Common magic bytes: PNG starts with "89 50 4E 47" (‰PNG), JPEG starts with "FF D8 FF," ZIP starts with "50 4B 03 04" (PK), PDF starts with "25 50 44 46" (%PDF). In CTFs, files often have wrong extensions or contain multiple embedded files!
